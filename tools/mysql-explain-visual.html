Tool 02

MySQL EXPLAIN Visual

EXPLAIN 결과를 분석해 실행 순서, 조인 관계, 인덱스 사용 여부와 비용 집중 구간을 시각적으로 보여줍니다.

Visual execution planWaiting for input

분석하는 항목

MySQL EXPLAIN 결과의 table, type, possible_keys, key, rows, filtered, Extra 값을 읽어 테이블 접근 순서와 조인 흐름을 요약합니다. full scan, temporary, filesort, 낮은 filtered 비율처럼 리뷰 중 먼저 봐야 할 신호를 화면에 모아 보여줍니다.

사용 방법

MySQL CLI, Workbench, Adminer 등에서 복사한 탭 구분 EXPLAIN 결과를 그대로 붙여넣으면 됩니다. 샘플 버튼으로 형식을 확인할 수 있고, 실행 계획 표와 시각 요약을 함께 보면서 인덱스 후보나 쿼리 분리 필요성을 검토할 수 있습니다.

보안과 한계

붙여넣은 실행 계획은 브라우저에서만 파싱되며 서버로 전송하지 않습니다. 실제 성능 판단에는 데이터 분포, MySQL 버전, 통계 최신성, 실행 시점의 캐시 상태가 영향을 주므로 이 도구는 병목 후보를 빠르게 좁히는 보조 도구로 사용하는 것이 좋습니다.
